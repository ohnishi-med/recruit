# 要件定義書 — クリニック求人専用ページ (recruit-site)

## 1. プロジェクト概要
`maps.mainichi.clinic` (医療機関MAP) と同様の自動デプロイ設計を採用し、クリニックの求人情報を魅力的かつモダンに伝える「求人専用特設ページ」を新規作成します。

---

## 2. 目的と背景
- **目的**: クリニックでの勤務を希望する求職者（医師、看護師、医療事務など）に対し、クリニックの魅力、理念、具体的な募集要項を分かりやすく伝える。
- **デプロイの簡易化**: GitHub へ Push するだけで、FTP経由で自動的に求人サイトが更新される環境を構築する。

---

## 3. 実装アプローチ
`maps` プロジェクトの構成を踏襲し、保守性とパフォーマンスに優れた静的WEBサイトとして構築します。

- **静的ホスティング**: HTML / Vanilla CSS / Vanilla JS による高速で信頼性の高いWEBサイト。
- **データ分離**: 求人要項などの変更が頻繁にあるデータは `src/data/jobs.json` に分離し、HTMLを直接書き換えなくてもJSONを更新するだけで求人票が追加・変更できるように設計。
- **自動化ビルド**: `tools/build.py` (Python) を使用して、ソースコードを `dist/` ディレクトリにコピー・最適化。
- **自動デプロイ**: GitHub Actions を使用して、`main` ブランチへの Push 時に FTP経由で自動的に本番サーバーへデプロイ。

---

## 4. UI/UX プレミアムデザイン方針
ユーザー体験をWOWさせるため、以下の最高品質デザインを取り入れます：

- **カラーパレット**: 医療としての信頼感を与える清潔なトーン（Deep Teal や Slate Blue などを基調とした美しいグラデーション）
- **タイポグラフィ**: システムフォントを避け、Google Fonts (Inter & Outfit / 筑紫ゴシック系フォント) を採用し、スタイリッシュで読みやすいテキスト配置。
- **マイクロアニメーション**: ボタンのホバー、カード要素のふわっとしたスクロール浮き上がり (Fade-in on scroll)、求人情報のインタラクティブなアコーディオン展開。
- **レスポンシブデザイン**: スマートフォンでの閲覧比率が非常に高いため、モバイル優先（Mobile First）設計。

---

## 5. ページ構成（案）
1. **ヒーローセクション (Hero Section)**: クリニックで働くイメージが湧くプレミアムなビジュアルと、キャッチコピー。
2. **院長メッセージ・理念 (Vision & Message)**: クリニックのビジョンと、求める人物像。
3. **数字で見るクリニック (Clinic by the Numbers)**: 残業時間、有給消化率、男女比、スタッフの平均年齢などを美しいインフォグラフィックスで紹介。
4. **募集要項 (Job Openings)**:
   - 職種別（医師、看護師、医療事務、受付など）のカードデザイン。
   - クリック/タップで展開するアコーディオン形式の詳細（業務内容、給与、勤務時間、休日、福利厚生など）。
5. **働く環境・福利厚生 (Benefits)**: 教育制度や充実した手当などを紹介。
6. **応募フロー & アクション (Apply & Flow)**: 応募から採用までの流れと、魅力的な「応募する」ボタン（外部フォームやLINEへのリンク）。

---

## 6. 決定事項および仕様
1. **本番URL**: `https://recruit.mainichi.clinic/`
2. **応募フロー (Googleフォーム & メール通知)**:
   - 応募者はサイト内のリンクから「Googleフォーム」へ遷移し、情報を入力。
   - Googleフォームの標準機能（または Google Apps Script (GAS)）を利用し、応募完了時に採用担当者へ自動でメール通知が届くように設定。
